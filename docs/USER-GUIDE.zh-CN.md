# style-memory-mcp 用户指南

这个工具用来帮 agent 保持稳定的说话和协作方式。它不是人格画像，也不是私人记忆库。

它记的是：

- 你喜欢 agent 怎么组织回答。
- 你喜欢什么样的语气边界。
- 哪些口癖、颜文字、方言、句尾习惯可以轻度参考。
- 哪些风格以后不要用。

它不应该记：

- 你的身份、住址、工作、账号、联系方式。
- 完整聊天记录。
- 密码、token、文件内容。
- “用户焦虑/内向/某某人格”这类心理或性格标签。

## 常用说法

你可以自然地对 agent 这样说：

- “感觉飘了，重新对齐一下。”
- “以后别这样。”
- “这个不是我的风格，忘掉。”
- “这个协作方式固定下来。”
- “少用颜文字。”
- “只读风格，先别继续学习。”
- “重新打开学习。”
- “看看你现在学了什么。”
- “给风格记忆打个分。”

agent 应该把这些话映射到 MCP 工具：

| 你说的话 | agent 应该做 |
|----------|--------------|
| “感觉飘了/重新对齐一下” | 调用 `get_style_brief` |
| “看看学了什么” | 调用 `list_style_habits` 和 `list_interaction_profile` |
| “这个不是我的风格/忘掉” | 调用 `forget_style_habit` 或 `forget_interaction_preference` |
| “这个固定下来” | 调用 `pin_style_habit` 或 `pin_interaction_preference` |
| “以后别这样/少用这个” | 删除相关项，或写入一个 `tone_boundary` 协作偏好 |
| “先别继续学习” | 调用 `set_learning_enabled(false)` |
| “重新打开学习” | 调用 `set_learning_enabled(true)` |
| “打个分/现在稳不稳” | 调用 `get_style_memory_score` |

## 推荐使用方式

一开始可以打开 learning，让它慢慢观察你。

当你觉得 agent 已经比较稳定：

1. 让它调用 `get_style_memory_score` 看分数和建议。
2. 让它调用 `review_style_habits` 和 `review_interaction_profile`。
3. 删除明显不对的项。
4. 固定很重要的协作偏好。
5. 用 `set_learning_enabled(false)` 进入只读复用模式。

之后新窗口启动时，它只需要调用 `get_style_brief`，就能拿到稳定风格。

## 什么时候需要重新对齐

下面这些情况适合重新拉 brief：

- 聊天很长以后，感觉回复不像之前稳定。
- 切换了大话题，比如从闲聊切到写代码、从写代码切到正式文档。
- agent 开始机械模仿你的口癖。
- agent 忘了你喜欢的协作顺序。
- 你准备让它写一段比较长、比较重要的内容。

你可以直接说：“感觉飘了，重新对齐一下。”

## 怎么避免越学越乱

- 不要让 agent 把每一句话都当成稳定偏好。
- 只有重复出现、跨场景出现、或者你明确背书的内容，才应该变 active。
- 颜文字、口癖、方言只轻度用，不要每句都用。
- 技术、正式、高风险场景优先清晰，不优先风格。
- 定期 review 候选项，删掉一次性的误判。

## 自用检查清单

如果你准备把它当真实用户使用，先确认：

- 新聊天会先调用 `get_style_brief`。
- 每条用户消息后会调用 `observe_user_message`。
- 长聊天会每 12-20 个用户回合重新调用 `get_style_brief`。
- “感觉飘了”会触发重新对齐。
- “以后别这样”会触发删除或新增边界偏好。
- `get_style_memory_score` 能告诉你是否需要 review。

这几项都做到，它就不是一次性提示词，而是一个可更新、可纠错、能长期复用的本地风格锚点。
